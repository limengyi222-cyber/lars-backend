# 如何给 LARS 增加一个省的适飞网格

采用「按需分省加载」（方案A）：只登记元数据，查询命中才加载，内存永远不爆。
新增一个省只需两步，**不影响现有省、不增加常驻内存**。

## 步骤

### 1. 把该省的 shapefile 转成 JSON

把 `<省>适飞网格.shp`（连同 .dbf/.shx/.prj）放好，运行：

```python
import shapefile, json
r = shapefile.Reader("路径/<省>适飞网格")   # 不带扩展名
grids = [[round(s.bbox[0],5), round(s.bbox[1],5),
          round(s.bbox[2],5), round(s.bbox[3],5)] for s in r.iterShapes()]
lons=[g[0] for g in grids]+[g[2] for g in grids]
lats=[g[1] for g in grids]+[g[3] for g in grids]
json.dump({"grids":grids,"count":len(grids),"crs":"EPSG:4326",
           "source":"<省>适飞网格 26版V2"},
          open("<省拼音>_grids.json","w"), separators=(',',':'))
print("格数:", len(grids))
print("bbox:", [min(lons),min(lats),max(lons),max(lats)])   # ← 复制这行给第2步
```

把生成的 `<省拼音>_grids.json` 放到后端根目录（和 gba_grids.json 同级）。

### 2. 在引擎里登记一行

编辑 `app/engines/airspace_engine.py` 的 `PROVINCES` 列表，加一行：

```python
_reg("省名", "<省拼音>_grids.json", [W, S, E, N], 格数),
```

- `[W,S,E,N]`、`格数` 用第1步打印出来的值
- 不要加 `pin=True`（只有广东是常驻主力）

### 3. 提交部署

```bash
git add <省拼音>_grids.json app/engines/airspace_engine.py
git commit -m "data: 新增<省>适飞网格"
git push
```

Render 自动部署后即生效。该省第一次被查询时才加载进内存，之后按 LRU 与其它省轮换。

## 注意

- **内存永远封顶**：无论登记多少省，常驻不超过 `_MAX_RESIDENT`(=3) 个省（含广东）。
- **仓库体积**：JSON 会进 git 仓库。若最终要并入全国十几个省（约 96MB），仓库会变大、
  部署变慢——那时应转对象存储/数据库外置（方案D），届时再议。
- 各省数据在 iCloud：`省低空安全公司/空域材料/26版无人机适飞空域网格V2/`
  （已有：上海 天津 宁夏 安徽 山西 江苏 江西 河北 河南 湖北 湖南 甘肃 贵州 重庆 陕西 青海 + 四川，共17省 shapefile）
