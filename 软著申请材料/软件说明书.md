## 一、简介

广东省低空适飞区域安全查询系统（以下简称"本系统"或"LARS"）是一套面向低空经济应用场景的在线安全评估平台。本系统依据中国民用航空局相关法规及 ICAO 国际标准，整合广东省 46,331 个适飞矩形网格数据、SRTM 30m 数字高程模型、实时 ADS-B 航班数据及气象数据，为无人机操控人员、航空运营单位和低空空域管理机构提供航线空域合规预审、碰撞风险量化评估、地形安全分析等核心功能，助力低空飞行活动安全、合规、高效开展。

本系统后端采用 Python 语言开发，以 FastAPI 异步框架对外提供 RESTful API 接口，部署于云端服务器，配套前端网页可通过浏览器访问（https://guangdong-airspace-safety.pages.dev/）。

## 四、设计需求

本系统面向以下应用场景设计：

1. **无人机飞行合规预审**：操控人员在执飞前查询航线是否穿越受限空域，获取可信评级结果，避免违规飞行。

2. **低空安全风险量化**：为航空运营单位提供 CREAM/TVR 等国际标准方法的碰撞风险数值，支持飞行安全报告编制。

3. **地形安全分析**：根据计划飞行高度和航线，自动分析沿线地形起伏，计算最低安全高度，预警 CFIT 风险。

4. **实时态势感知**：通过 ADS-B 数据获取实时空域态势，识别热点区域和拥挤航段，支持空域管理决策。

5. **历史数据追溯**：记录每次评估结果，支持按用户、按时间查询，满足运营合规存档需求。

软件具备以下核心功能需求：

1. 具备适飞区域查询功能：接收航路点坐标，返回适飞/受限/无数据判定结果
2. 具备多维度风险计算功能：支持碰撞风险、垂直风险、地形风险、空域复杂度等量化评估
3. 具备实时数据接入功能：对接 ADS-B、气象等外部数据源
4. 具备用户管理功能：支持注册、登录、鉴权、历史记录查询
5. 具备速率保护功能：防止接口滥用，保障服务稳定

### 5.2 适飞区域查询功能

适飞区域查询是本系统的核心功能，查询流程如下：

```
接收 API 请求 POST /api/v1/airspace/route-check
      ↓
参数解析：航路点列表 [{lat, lon}, ...]
      ↓
检查是否已加载适飞网格
  ├── 未加载 → 读取 gba_grids.json → 建立空间桶索引（0.1° 分桶）
  └── 已加载 → 直接使用单例缓存
      ↓
沿航线等距插值 N 个采样点（默认 80 个）
      ↓
逐点判定（O(1) 空间索引查询）：
  ├── 在广东范围内 + 命中网格 → flyable（适飞）
  ├── 在广东范围内 + 未命中  → restricted（受限）
  └── 超出广东范围            → no_data（无数据）
      ↓
识别连续受限段、无数据段
计算受限比例、覆盖率
      ↓
综合评级：
  ├── restricted=0 且 no_data=0 → PASS
  ├── restricted=0 且 no_data>0 → WARNING
  └── restricted>0              → RESTRICTED
      ↓
返回 JSON：profile / summary / segments / verdict
```

适飞网格加载与空间索引建立的核心代码：

```python
BUCKET_SIZE = 0.1  # 度，约 11km

def _load_grids():
    global _GRIDS, _BUCKETS, _LOADED
    if _LOADED:
        return
    with open(grid_path, "r") as f:
        data = json.load(f)
    _GRIDS = data["grids"]
    # 构建空间桶索引
    for idx, g in enumerate(_GRIDS):
        min_lon, min_lat, max_lon, max_lat = g
        bi_start = int(min_lon / BUCKET_SIZE)
        bi_end   = int(max_lon / BUCKET_SIZE)
        bj_start = int(min_lat / BUCKET_SIZE)
        bj_end   = int(max_lat / BUCKET_SIZE)
        for bi in range(bi_start, bi_end + 1):
            for bj in range(bj_start, bj_end + 1):
                key = (bi, bj)
                if key not in _BUCKETS:
                    _BUCKETS[key] = []
                _BUCKETS[key].append(idx)
    _LOADED = True
```

单点查询的核心代码：

```python
def _point_in_grids(lon: float, lat: float) -> str:
    # 超出广东覆盖范围
    if not (GD_LON_MIN <= lon <= GD_LON_MAX and
            GD_LAT_MIN <= lat <= GD_LAT_MAX):
        return "no_data"
    bi = int(lon / BUCKET_SIZE)
    bj = int(lat / BUCKET_SIZE)
    candidates = _BUCKETS.get((bi, bj), [])
    for idx in candidates:
        g = _GRIDS[idx]
        if g[0] <= lon <= g[2] and g[1] <= lat <= g[3]:
            return "flyable"
    return "restricted"
```

### 5.3 CREAM 三维碰撞风险计算

CREAM 计算流程基于 ICAO Doc 9689 + NTU ATMRI 方法论，核心为三维碰撞风险乘积模型：

**垂直重叠概率 Pz(Sz)：**

系统采用 Laplace（高度表系统误差 ASE）与 Normal（实际高度偏差 AAD）的解析卷积结果（Reed & Jorgensen 2004 正态-拉普拉斯分布）：

```
f_TVE(z) = exp(σ²/2b²) / (4b) × [exp(-z/b)·erfc(-α) + exp(z/b)·erfc(β)]
其中：α = (z/σ - σ/b) / √2，β = (z/σ + σ/b) / √2
```

TVE 分布计算代码（cream_engine.py，节选）：

```python
def _tve_pdf(z: float, sigma_aad: float, b_ase: float) -> float:
    s, b = float(sigma_aad), float(b_ase)
    alpha = (z / s - s / b) / np.sqrt(2.0)
    beta  = (z / s + s / b) / np.sqrt(2.0)
    log_scale = s ** 2 / (2.0 * b ** 2)
    term1 = np.exp(log_scale - z / b) * float(erfc(-alpha))
    term2 = np.exp(log_scale + z / b) * float(erfc(beta))
    return float(np.clip((term1 + term2) / (4.0 * b), 0.0, None))
```

**总三维碰撞风险：**

```
CR_3D = Px(Sx) × Py(Sy) × Pz(Sz) × K_xy × K_z × N_ac
```

### 5.4 地形 CFIT 风险分析

地形分析流程如下：

```
接收航路点列表及计划飞行高度（AGL）
      ↓
沿航线等距插值 60 个采样点
      ↓
批量查询 OpenTopoData SRTM 30m API（每批最多 100 点）
      ↓
逐点计算：
  flight_amsl = elev + altitude_agl  （AGL → AMSL 转换）
  clearance   = altitude_agl - OBS_BUFFER - MOC
  p_cfit      = 1 - Φ(clearance / σ_alt)  （高斯分布尾概率）
      ↓
统计最高地形高度、最小超障余度
计算推荐最低安全高度 MSA = OBS_BUFFER + MOC
      ↓
评级：
  ├── min_clearance ≥ MOC 且 p_cfit < 1e-7 → PASS
  ├── min_clearance ≥ 0 且 p_cfit < 1e-3  → WARNING
  └── 其他                                 → FAIL
      ↓
返回地形剖面、风险评级、关键点列表
```

核心参数默认值：
- MOC（最低超障余度）：50 m（ICAO UAV 建议值）
- OBS_BUFFER（障碍物缓冲）：50 m
- σ_alt（GPS 垂直误差 1σ）：15 m

### 5.5 总垂直风险（TVR）计算

总垂直风险叠加三种 LHD 情形：

```
Naz_total = Naz_NWL + Naz_CLD + Naz_WL
```

- **Naz_NWL**：技术误差情形，基于混合 AAD 分布（正常飞行 s1 与异常飞行 s2 的加权混合）
- **Naz_CLD**：高度层偏差事件（Controlled Level Deviation），依据事件次数与持续时间比例
- **Naz_WL**：错误高度层事件（Wrong Level），依据 0.538 修正系数

密码哈希代码（auth_engine.py）：

```python
def _hash_password(password: str) -> str:
    salt = secrets.token_hex(16)
    dk = hashlib.pbkdf2_hmac(
        "sha256", password.encode(), salt.encode(), 260000
    )
    return f"pbkdf2:{salt}:{dk.hex()}"
```

登录成功后签发随机 Session Token（secrets.token_hex(32)），有效期 30 天，存储于数据库。后续请求通过 `Authorization: Bearer <token>` 鉴权。

### 5.8 速率限制机制

系统内置进程级 IP 速率限制器，防止接口滥用：

```python
def _rate_check(request: Request, limit: int = 30, window: int = 60):
    ip = request.client.host
    now = time.monotonic()
    cutoff = now - window
    with _RL_LOCK:
        _RL_BUCKETS[ip] = [t for t in _RL_BUCKETS[ip] if t > cutoff]
        if len(_RL_BUCKETS[ip]) >= limit:
            raise HTTPException(
                status_code=429,
                detail=f"请求过于频繁，{window}秒内最多{limit}次"
            )
        _RL_BUCKETS[ip].append(now)
```

各类接口速率限制：
- 计算类接口（CREAM/TVR）：每 IP 每分钟 30 次
- 地形分析接口：每 IP 每分钟 15 次
- 热点/网络/复杂度接口：每 IP 每分钟 10 次

## 八、安全设计

1. **密码安全**：采用 PBKDF2-HMAC-SHA256，随机 salt，迭代 260,000 次，符合 NIST SP 800-63B 建议。
2. **Session 安全**：Token 采用 `secrets.token_hex(32)` 生成（256 位熵），存储于数据库，有效期 30 天，退出即清除。
3. **CORS 白名单**：严格限制跨域访问来源，仅允许指定域名。
4. **速率限制**：进程级 IP 速率限制，防止 DoS 攻击及接口滥用。
5. **时序攻击防护**：密码比对使用 `secrets.compare_digest` 恒时比较。
6. **参数校验**：所有 API 输入通过 Pydantic BaseModel 自动校验类型和范围。
